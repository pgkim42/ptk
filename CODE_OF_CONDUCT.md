# Code of Conduct

PTK is small, but the project should still be easy to participate in.

## Expected Behavior

- Be kind.
- Assume good faith.
- Stay specific when reporting bugs or reviewing changes.
- Critique code and behavior, not people.
- Avoid sharing private machine data, secrets, or credentials.

## Unacceptable Behavior

- Harassment or personal attacks.
- Publishing someone else's private information.
- Repeatedly pushing unsafe process-termination behavior after it has been
  rejected.
- Using issues or pull requests for spam.

## Reporting Concerns

Use GitHub reporting tools for platform abuse. For project-specific concerns,
open a GitHub issue without sensitive personal details, or ask the maintainer
for a private channel.

## Maintainer Response

The maintainer may edit, hide, or remove comments and may close issues or pull
requests that make the project less safe or less useful to maintain.

## Scope

This code of conduct applies to PTK issues, pull requests, discussions, and
other project-maintained spaces.
