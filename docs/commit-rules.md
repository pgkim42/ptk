# 커밋 규칙

이 프로젝트는 아래 커밋 규칙을 반드시 따른다.

## 1) 커밋 컨벤션 준수

- Conventional Commits 형식을 사용한다.
- 기본 형식:
  - `<prefix>(<scope>): <제목>`
- `scope`는 필요할 때만 사용한다.

허용 prefix 예시:
- `feat`
- `fix`
- `refactor`
- `docs`
- `test`
- `chore`
- `build`
- `ci`
- `perf`
- `style`

## 2) Prefix는 영어

- prefix는 반드시 영어 소문자를 사용한다.
- 예: `feat`, `fix`, `docs`

## 3) 제목과 본문은 한국어

- 커밋 제목(subject)은 한국어로 작성한다.
- 커밋 본문(body)도 한국어로 작성한다.

## 4) 50/72 규칙 준수

- 제목은 50자 이내로 작성한다.
- 본문 각 줄은 72자 이내로 줄바꿈한다.
- 제목과 본문 사이에는 한 줄을 비운다.

## 5) 권장 템플릿

```text
<prefix>(<scope>): <50자 이내 한국어 제목>

<72자 이내 줄바꿈을 지킨 한국어 본문>
<변경 이유, 영향 범위, 테스트 결과를 간단히 작성>
```

## 6) 예시

```text
feat(scanner): 자동 감시 주기 설정 기능 추가

기본 감시 주기를 3초로 고정하고 사용자가 토글로 켜고 끌 수
있도록 UI를 수정했다. 수동 스캔 동작은 기존과 동일하게 유지했다.
```
